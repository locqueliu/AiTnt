# AiTnt

[English Version](./README.md)

AiTnt 是我自己一直在捏的一套本地桌面创作工作站。

最开始做它，纯粹是因为我受不了创作流程太碎。图像生成在一个地方，视频生成在另一个地方，提示词一堆 txt，素材又散在各种文件夹里，想接着上次做都得先花一会儿找东西。AiTnt 说白了就是想把这些东西收回到一个顺手的工作区里，打开就能继续干。

## 现在主要在做的几块

- 图像工作区：文生图、图生图和输出整理放一起
- 视频工作区：生成任务、队列和导出流程放一起
- 资源库：整理提示词、模板和能反复用的素材
- 节点画布：把工作流导入、导出和整理收在一起
- 快捷应用：把商品图、风格化这类高频场景单独做成入口
- 设置中心：统一管供应商、模型、目录、语言和工作区配置

## 技术栈

- Electron
- React
- TypeScript
- Vite
- Zustand
- XYFlow
- dnd-kit

## 本地开发

安装依赖：

```bash
npm install
```

启动开发环境：

```bash
npm run dev
```

构建桌面应用：

```bash
npm run build
```

## 我做这个项目时比较认的几件事

- 生成和整理别分家
- 好用的工作流得能沉淀下来
- 软件打开以后最好别让人重新找状态
- 视觉输出、提示词资产和节点逻辑都算正经内容

## 目录说明

- `src/` 应用源码
- `scripts/` 构建辅助脚本
- `build/` 打包资源
- `public/` 静态资源

## 最后一句

这个仓库会跟着我的实际使用方式一起长。很多功能最开始都只是“先救一下当下这个需求”，但只要用着顺、复用率高，后面我一般都会把它慢慢磨成正式模块。
